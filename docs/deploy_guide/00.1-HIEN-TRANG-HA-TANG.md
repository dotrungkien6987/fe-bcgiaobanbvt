## ❓ VẤN ĐỀ CẦN GIẢI QUYẾT

### **Tại sao cần HTTPS?**

```
HTTP (không có S):
┌─────────────────────────────────────────────────────────┐
│  Browser ──── "password123" ────→ Server               │
│                    ↑                                    │
│           Ai cũng đọc được! 👁️                          │
│           (WiFi, hacker, v.v.)                          │
└─────────────────────────────────────────────────────────┘

HTTPS (có S = Secure):
┌─────────────────────────────────────────────────────────┐
│  Browser ══ "a7f3b2..." (mã hóa) ══→ Server            │
│                    ↑                                    │
│           Không ai đọc được! 🔒                         │
│           Chỉ browser và server hiểu                    │
└─────────────────────────────────────────────────────────┘

Ứng dụng bệnh viện BẮT BUỘC dùng HTTPS vì:
├─ Có thông tin bệnh nhân (nhạy cảm)
├─ Có mật khẩu đăng nhập
├─ Browsers hiện đại chặn HTTP (không an toàn)
├─ PWA/Service Worker yêu cầu HTTPS
└─ Push Notifications (FCM) yêu cầu HTTPS
```

### **Vấn đề: Không có port 443!**

```
Để làm HTTPS cần:
1. Port 443 được mở từ Internet đến server của bạn
2. SSL Certificate (chứng chỉ bảo mật)

Hiện tại:
├─ Port 443 đã bị Máy B chiếm ❌
├─ Bạn không kiểm soát Máy B ❌
├─ IT chưa cấp port 443 cho bạn ❌
└─ → KHÔNG THỂ làm HTTPS trực tiếp được!

Giải pháp tạm thời: CLOUDFLARE TUNNEL ✅
├─ Không cần port 443
├─ HTTPS miễn phí
├─ Deploy được ngay
└─ Dùng trong khi chờ có Máy C
```

   "Tại sao không cần mở port 443?"

   → Bình thường: User kết nối VÀO server (inbound) cần port mở

                  → Cloudflare đóng vai trò trung gian

### **Ưu điểm & Nhược điểm:**

```
✅ ƯU ĐIỂM:                              ⚠️ NHƯỢC ĐIỂM:
├─ Không cần port 80/443                 ├─ Latency cao hơn (qua CF)
├─ HTTPS tự động, miễn phí              ├─ WebSocket có giới hạn
├─ DDoS protection miễn phí             │   (nhiều users đồng thời)
├─ Deploy nhanh (1-2 ngày)              ├─ Phụ thuộc Cloudflare
├─ Không cần cấu hình SSL               │   (nếu CF down → app down)
├─ CDN tích hợp                         └─ Không kiểm soát 100%
└─ Phù hợp giai đoạn đầu
```

║                      SWITCH GIỮA 2 GIAI ĐOẠN                             ║
╠═══════════════════════════════════════════════════════════════════════════╣
║                                                                           ║

║                                                                           ║

║     │  (Giai đoạn 1)  │  SWITCH   │  (Giai đoạn 2)  │                    ║
║     └─────────────────┘           └─────────────────┘                    ║
║                                                                           ║
║  Khi nào switch sang CF Tunnel (tạm thời):                               ║

║                                                                           ║
║  Khi nào switch sang Nginx:                                               ║
║  ├─ Bình thường (production)                                             ║

║  └─ WebSocket nhiều users                                                 ║
║                                                                           ║
║  Thời gian switch: ~15-30 phút                                           ║
║  Downtime: ~5 phút (DNS propagation)                                     ║
║                                                                           ║
║  Chi tiết: Xem File 24-SWITCH-GUIDE.md                                   ║
║                                                                           ║
╚═══════════════════════════════════════════════════════════════════════════╝

## ✅ CHECKLIST TRƯỚC KHI BẮT ĐẦU

```
Trước khi bắt đầu Giai đoạn 1, đảm bảo bạn có:

□ Máy chủ A (192.168.1.243) đã cài Ubuntu Server
□ Có tài khoản user với quyền sudo
□ Có thể SSH vào server từ máy tính cá nhân
□ Đã mua domain tại TenTen.vn
□ Đã tạo tài khoản Cloudflare (miễn phí)
□ Đã có source code ứng dụng (React + Node.js)
□ Notepad để ghi chú passwords, keys

Không cần ngay:
○ Máy chủ C (sẽ có sau ~2 tuần)
○ Port 80/443 (sẽ có khi có Máy C)
○ SSL Certificate (CF Tunnel lo)
```
